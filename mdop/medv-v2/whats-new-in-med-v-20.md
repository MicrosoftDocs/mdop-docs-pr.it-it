# <span data-ttu-id="02ff9-103">Novità in MED-V 2.0</span><span class="sxs-lookup"><span data-stu-id="02ff9-103">What's New in MED-V 2.0</span></span>

## <span data-ttu-id="02ff9-106">Modifiche della funzionalità standard</span><span class="sxs-lookup"><span data-stu-id="02ff9-106">Changes in Standard Functionality</span></span>


<span data-ttu-id="02ff9-107">In questa sezione vengono illustrate le aree principali in cui è cambiata la funzionalità MED-V 2,0.</span><span class="sxs-lookup"><span data-stu-id="02ff9-107">This section discusses the key areas where MED-V 2.0 functionality has changed.</span></span>

### <span data-ttu-id="02ff9-108">Creazione di aree di lavoro MED-V</span><span class="sxs-lookup"><span data-stu-id="02ff9-108">MED-V Workspace Creation</span></span>

<span data-ttu-id="02ff9-109">Il disco rigido virtuale usato per l'area di lavoro MED-V è ora creato in Windows Virtual PC.</span><span class="sxs-lookup"><span data-stu-id="02ff9-109">The virtual hard disk used for the MED-V workspace is now created in Windows Virtual PC.</span></span> <span data-ttu-id="02ff9-110">I metodi usati per creare l'area di lavoro MED-V includono installare Windows XP SP3, aggiornare il sistema operativo e prepararlo per essere gestito tramite l'infrastruttura di gestione del software.</span><span class="sxs-lookup"><span data-stu-id="02ff9-110">The methods that are used to create the MED-V workspace include installing Windows XP SP3, updating the operating system, and preparing it to be managed through software management infrastructure.</span></span>

<span data-ttu-id="02ff9-111">Le funzionalità di gestione offline e di trasferimento trim sono state rimosse, oltre alle funzionalità di crittografia e compressione dell'area di lavoro MED-V proprietarie.</span><span class="sxs-lookup"><span data-stu-id="02ff9-111">The offline management and trim transfer functionality were removed, in addition to the proprietary MED-V workspace encryption and compression functionality.</span></span> <span data-ttu-id="02ff9-112">Quando si crea un'area di lavoro MED-V, un amministratore deve preparare e configurare le applicazioni e gli strumenti di gestione appropriati nell'immagine invece di usare lo strumento di preparazione della macchina virtuale fornito in MED-V 1,0.</span><span class="sxs-lookup"><span data-stu-id="02ff9-112">When you create a MED-V workspace, an administrator should prepare and configure appropriate applications and management tools in the image instead of using the virtual machine preparation tool that is provided in MED-V 1.0.</span></span>

<span data-ttu-id="02ff9-113">L'uso di Sysprep nell'immagine MED-V è ora obbligatorio e convalidato durante la creazione dell'area di lavoro MED-V.</span><span class="sxs-lookup"><span data-stu-id="02ff9-113">Running Sysprep on the MED-V image is now required and validated during the packaging of the MED-V workspace.</span></span> <span data-ttu-id="02ff9-114">Packager area di lavoro MED-V offre un'interfaccia utente grafica (GUI) che guida l'amministratore attraverso il processo di creazione di pacchetti.</span><span class="sxs-lookup"><span data-stu-id="02ff9-114">The MED-V Workspace Packager provides a graphical user interface (GUI) that guides the administrator through the packaging process.</span></span> <span data-ttu-id="02ff9-115">La console di MED-V 1,0 è stata rimossa insieme alla funzionalità di gestione delle immagini, alla gestione dei profili di area di lavoro MED-V e all'esigenza di organizzare e crittografare le aree di lavoro MED-V.</span><span class="sxs-lookup"><span data-stu-id="02ff9-115">The console from MED-V 1.0 was removed together with the functionality of managing images, managing MED-V workspace profiles, and the requirement to stage and encrypt MED-V workspaces.</span></span>

### <span data-ttu-id="02ff9-116">Distribuzione di area di lavoro MED-V</span><span class="sxs-lookup"><span data-stu-id="02ff9-116">MED-V Workspace Deployment</span></span>

<span data-ttu-id="02ff9-117">Per distribuire un'area di lavoro MED-V, un amministratore è ora in grado di sfruttare gli strumenti di distribuzione del software elettronici.</span><span class="sxs-lookup"><span data-stu-id="02ff9-117">To deploy a MED-V workspace, an administrator is now able to take advantage of their electronic software distribution tools.</span></span> <span data-ttu-id="02ff9-118">Il metodo client-pull disponibile in MED-V 1,0 è stato rimosso e l'area di lavoro MED-V viene ora recapitata tramite metodi esterni a MED-V.</span><span class="sxs-lookup"><span data-stu-id="02ff9-118">The client-pull method available in MED-V 1.0 was removed and the MED-V workspace is now delivered by using methods outside MED-V.</span></span> <span data-ttu-id="02ff9-119">Gli amministratori possono gestire le aree di lavoro MED-V come farebbero con qualsiasi altro pacchetto di applicazioni e possono pianificare distribuzioni e installazioni di MED-V usando gli strumenti e i processi esistenti.</span><span class="sxs-lookup"><span data-stu-id="02ff9-119">Administrators can treat MED-V workspaces as they would any other application package and can schedule deployments and installations of MED-V by using their existing tools and processes.</span></span> <span data-ttu-id="02ff9-120">Le installazioni di MED-V possono essere distribuite in modo invisibile all'utente e possono essere gestite facilmente in un'infrastruttura di distribuzione del software esistente.</span><span class="sxs-lookup"><span data-stu-id="02ff9-120">MED-V installations can be deployed silently and can easily be managed inside an existing software distribution infrastructure.</span></span>

### <span data-ttu-id="02ff9-121">Gestione area di lavoro MED-V</span><span class="sxs-lookup"><span data-stu-id="02ff9-121">MED-V Workspace Management</span></span>

<span data-ttu-id="02ff9-122">L'area di lavoro MED-V in MED-V 2,0 si basa su un disco rigido virtuale di Windows Virtual PC.</span><span class="sxs-lookup"><span data-stu-id="02ff9-122">The MED-V workspace in MED-V 2.0 is based on a Windows Virtual PC virtual hard disk.</span></span> <span data-ttu-id="02ff9-123">MED-V ha esteso le funzionalità fornite da Windows Virtual PC migliorandone l'esperienza senza richiedere la crittografia o strumenti speciali per accedere all'area di lavoro MED-V.</span><span class="sxs-lookup"><span data-stu-id="02ff9-123">MED-V has extended the capabilities that Windows Virtual PC provides by improving the seamless experience without requiring encryption or special tools to access the MED-V workspace.</span></span>

<span data-ttu-id="02ff9-124">Dopo la distribuzione di MED-V in una workstation, l'area di lavoro MED-V può essere aperta in modalità schermo intero tramite Windows Virtual PC.</span><span class="sxs-lookup"><span data-stu-id="02ff9-124">After MED-V is deployed to a workstation, the MED-V workspace can be opened in full-screen mode by using Windows Virtual PC.</span></span> <span data-ttu-id="02ff9-125">Questa nuova funzionalità ha rimosso il requisito per i criteri che impostano una preferenza per le modalità seamless o a schermo intero e ha anche rimosso la necessità di forzare la visualizzazione a schermo intero per la diagnostica e la risoluzione dei problemi.</span><span class="sxs-lookup"><span data-stu-id="02ff9-125">This new functionality removed the requirement for policies that set a preference for seamless or full-screen modes and also removed the need to force full-screen for diagnostics and troubleshooting.</span></span>

<span data-ttu-id="02ff9-126">Le applicazioni di pubblicazione nell'area di lavoro MED-V non vengono più eseguite con i profili e immettendo manualmente il percorso delle applicazioni.</span><span class="sxs-lookup"><span data-stu-id="02ff9-126">Publishing applications to the MED-V workspace is no longer performed with profiles and by manually entering the path to applications.</span></span> <span data-ttu-id="02ff9-127">Si verifica invece automaticamente quando le applicazioni sono installate nel guest.</span><span class="sxs-lookup"><span data-stu-id="02ff9-127">Instead, it occurs automatically as applications are installed on the guest.</span></span> <span data-ttu-id="02ff9-128">Viene rimosso il repository di immagini centrale che include versioni delle immagini recapitate tramite il trasferimento trim.</span><span class="sxs-lookup"><span data-stu-id="02ff9-128">The central image repository that included versions of the images that were delivered through trim transfer is removed.</span></span> <span data-ttu-id="02ff9-129">Invece, MED-V consente agli amministratori di gestire l'area di lavoro MED-V come farebbero con un computer fisico, lasciando che le applicazioni e gli aggiornamenti vengano distribuiti senza la complessità di un'infrastruttura MED-V dedicata.</span><span class="sxs-lookup"><span data-stu-id="02ff9-129">Instead, MED-V enables administrators to manage the MED-V workspace as they would a physical computer, by letting applications and updates be distributed without the complexity of a dedicated MED-V infrastructure.</span></span>

### <span data-ttu-id="02ff9-132">Creazione di aree di lavoro MED-V</span><span class="sxs-lookup"><span data-stu-id="02ff9-132">MED-V Workspace Creation</span></span>

<span data-ttu-id="02ff9-133">Le aree di lavoro MED-V devono essere create usando Virtual PC Windows.</span><span class="sxs-lookup"><span data-stu-id="02ff9-133">MED-V workspaces must be created by using Windows Virtual PC.</span></span> <span data-ttu-id="02ff9-134">È necessario eseguire la migrazione delle immagini Virtual PC 2007 esistenti.</span><span class="sxs-lookup"><span data-stu-id="02ff9-134">Existing Virtual PC 2007 images must be migrated.</span></span> <span data-ttu-id="02ff9-135">Lo strumento di preparazione della macchina virtuale non è incluso in MED-V 2,0 e gli amministratori devono configurare, aggiornare e ottimizzare le proprie immagini in base al file della Guida di MED-V 2,0.</span><span class="sxs-lookup"><span data-stu-id="02ff9-135">The virtual machine Prep tool is not included in MED-V 2.0 and administrators should configure, update, and optimize their images according to the MED-V 2.0 Help file.</span></span> <span data-ttu-id="02ff9-136">L'esecuzione di Sysprep nell'immagine MED-V è un passaggio obbligatorio e deve essere eseguita prima della creazione di pacchetti.</span><span class="sxs-lookup"><span data-stu-id="02ff9-136">Running Sysprep on the MED-V image is a required step and must be performed before packaging.</span></span>

### <span data-ttu-id="02ff9-137">Pacchetto area di lavoro MED-V</span><span class="sxs-lookup"><span data-stu-id="02ff9-137">MED-V Workspace Packaging</span></span>

<span data-ttu-id="02ff9-138">Windows PowerShell è la base del Packager area di lavoro MED-V.</span><span class="sxs-lookup"><span data-stu-id="02ff9-138">Windows PowerShell is the foundation of the MED-V Workspace Packager.</span></span> <span data-ttu-id="02ff9-139">Questa funzionalità sostituisce alcune abilità e funzionalità precedenti della console che gestiscono le funzioni centralizzate di MED-V.</span><span class="sxs-lookup"><span data-stu-id="02ff9-139">This functionality replaces some former console abilities and functionality that managed centralized functions of MED-V.</span></span> <span data-ttu-id="02ff9-140">Il Packager area di lavoro MED-V consente semplicemente di creare un pacchetto del disco rigido virtuale con le impostazioni e l'immagine appropriate in modo che possano essere facilmente distribuite dagli amministratori.</span><span class="sxs-lookup"><span data-stu-id="02ff9-140">The MED-V Workspace Packager merely packages the virtual hard disk with the appropriate settings and image so that it can be easily deployed by administrators.</span></span> <span data-ttu-id="02ff9-141">Le funzionalità avanzate vengono fornite tramite Windows PowerShell.</span><span class="sxs-lookup"><span data-stu-id="02ff9-141">Advanced features are provided by using Windows PowerShell.</span></span>

### <span data-ttu-id="02ff9-142">Distribuzione dell'area di lavoro MED-V</span><span class="sxs-lookup"><span data-stu-id="02ff9-142">MED-V Workspace Distribution</span></span>

<span data-ttu-id="02ff9-143">L'infrastruttura server dedicata non è più necessaria per MED-V 2,0 e il metodo pull client per la distribuzione delle aree di lavoro MED-V è stato rimosso.</span><span class="sxs-lookup"><span data-stu-id="02ff9-143">Dedicated server infrastructure is no longer required for MED-V 2.0 and the client pull method to deploy MED-V workspaces was removed.</span></span> <span data-ttu-id="02ff9-144">Le aree di lavoro MED-V vengono ora distribuite con l'infrastruttura di distribuzione del software elettronica e possono essere archiviate in condivisioni comuni usate per altri pacchetti di installazione.</span><span class="sxs-lookup"><span data-stu-id="02ff9-144">MED-V workspaces are now deployed using your electronic software distribution infrastructure and can be stored on common shares that are used for other installation packages.</span></span>

### <span data-ttu-id="02ff9-149">Pubblicazione di applicazioni</span><span class="sxs-lookup"><span data-stu-id="02ff9-149">Application Publishing</span></span>

<span data-ttu-id="02ff9-150">Gli amministratori possono installare le applicazioni nell'immagine MED-V prima della creazione del pacchetto, dopo la distribuzione dell'area di lavoro MED-V o tramite una combinazione di entrambi.</span><span class="sxs-lookup"><span data-stu-id="02ff9-150">Administrators can install applications on the MED-V image either before packaging, after the MED-V workspace is deployed, or by using a combination of both.</span></span> <span data-ttu-id="02ff9-151">MED-V non esamina più i criteri dell'area di lavoro MED-V per pubblicare le applicazioni, ma fa riferimento a ciò che viene effettivamente installato nel guest.</span><span class="sxs-lookup"><span data-stu-id="02ff9-151">MED-V no longer examines MED-V workspace policy to publish applications, but instead refers to what is actually installed on the guest.</span></span> <span data-ttu-id="02ff9-152">Mentre le applicazioni sono installate nel guest, vengono rilevate e pubblicate automaticamente nel menu **Start** dell'host e sono pronte per essere avviate dall'utente finale.</span><span class="sxs-lookup"><span data-stu-id="02ff9-152">As applications are installed on the guest, they are automatically detected and published to the host **Start** menu and are ready to be started by the end user.</span></span>

### <span data-ttu-id="02ff9-153">Reindirizzamento URL</span><span class="sxs-lookup"><span data-stu-id="02ff9-153">URL Redirection</span></span>

<span data-ttu-id="02ff9-154">MED-V 2,0 offre un reindirizzamento degli indirizzi Web host-to-Guest in base ai criteri configurati e gestiti dall'amministratore.</span><span class="sxs-lookup"><span data-stu-id="02ff9-154">MED-V 2.0 provides seamless host-to-guest web address redirection based on the policies configured and managed by the administrator.</span></span> <span data-ttu-id="02ff9-155">Dopo che un URL viene reindirizzato al browser Guest, l'esperienza predefinita consiste nel tentativo di limitare l'utente al sito reindirizzato.</span><span class="sxs-lookup"><span data-stu-id="02ff9-155">After a URL is redirected to the guest browser, the default experience is to attempt to limit the user to that redirected site.</span></span> <span data-ttu-id="02ff9-156">Questo riduce a icona le attività di esplorazione che un utente può eseguire che non sono intese dall'amministratore.</span><span class="sxs-lookup"><span data-stu-id="02ff9-156">This minimizes the browsing activities that a user can perform that are not intended by the administrator.</span></span> <span data-ttu-id="02ff9-157">Il reindirizzamento del browser Guest-to-host è stato rimosso.</span><span class="sxs-lookup"><span data-stu-id="02ff9-157">Guest-to-host browser redirection was removed.</span></span>
